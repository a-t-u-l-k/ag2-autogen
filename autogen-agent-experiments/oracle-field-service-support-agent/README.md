# Oracle Field Service Support Agent

This demo models an Oracle Field Service support workflow with chat, knowledge lookup, and sentiment-aware routing.

Files:
- `server.py`: main backend service.
- `oracle-field-service-agent.py`: primary support-agent implementation.
- `oracle-field-service-agent2.py`: alternate implementation variant.
- `index.html`: main UI entry point.
- `static/index.html`: static client copy used by the app.

Notes:
- Some flows expect local helper services or placeholder endpoints to be configured.
